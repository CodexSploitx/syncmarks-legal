Privacy Policy – SyncMarks

Last updated: 2025-11-27

Controller: SyncMarks By CodexSploitx

1. Information We Collect

SyncMarks collects the minimum information necessary to provide the service.

1.1 Data Provided by the User

  • Email
  • Username
  • Password (stored exclusively as an irreversible hash)

1.2 Data Generated by the User within the Extension

  • Bookmarks created by the user (title, URL, assigned folder)
  • Site favicon URL (when available)
  • Custom bookmark folders

1.3 Temporary Technical Data

We may temporarily store the IP address solely to:

  • Protect against abuse
  • Detect bots
  • Enforce rate limits
  • Block IP addresses with malicious activity

These data are automatically deleted and not kept permanently.

2. How We Use the Information

The information collected is used only to:

Provide SyncMarks features

  • Registration and sign-in
  • Synchronization of bookmarks across devices
  • Management of folders and collections
  • Fast loading of bookmarks from local cache

Protect the platform

  • Prevent abuse and attacks
  • Maintain the stability of our API services
  • Control unauthorized access

SyncMarks does not use data for advertising and does not sell it.

3. Where Information Is Stored

3.1 Data in the Extension

User data is stored locally in:

  • chrome.storage.local — bookmarks and folders
  • chrome.storage.sync — minimal user data for quick access

These data remain until the user signs out or reinstalls the extension.

3.2 Data on the Server

The following data are stored in our secure database (MongoDB):

  • Email
  • Username
  • Password (hash)
  • Synchronized bookmarks and folders

All data are transmitted over encrypted HTTPS.

4. Synchronization

SyncMarks offers two types of synchronization:

4.1 Automatic Synchronization

  • Occurs in the background when the user signs in
  • Occurs when the extension detects new changes from the server

4.2 Manual Synchronization

The user can trigger synchronization at any time from the extension.

5. Sharing Data with Third Parties

SyncMarks does not share, sell, trade, or transfer user data to third parties.

We do not use external analytics or advertising services that collect personal data.

6. Security

We are committed to protecting user information through:

  • Irreversible password hashing
  • Secure storage in MongoDB
  • Encrypted HTTPS connection
  • Protection against data injection
  • Rate limits and firewalls
  • Isolation of local storage per user (namespaces)

Access to data is strictly controlled.

7. User Rights

  • Complete deletion of their account
  • Deletion of their bookmarks stored on the server
  • Export of their bookmarks
  • Revocation of access and sign-out on all devices

For inquiries, contact: (your support email)

8. Data Deletion

If the user deletes their account:

  • All their information is permanently removed from the servers.
  • No copies of bookmarks, folders, or credentials are retained.

9. Changes to This Policy

SyncMarks may update this policy due to improvements or legal changes.

Users will be notified within the extension and on our official website.
